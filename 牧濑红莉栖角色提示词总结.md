# 牧濑红莉栖角色提示词总结

本文档基于 `E:\2025llm\amadeus-system-new` 项目源码整理，范围只包括项目中直接或间接用于构建“牧濑红莉栖/Kurisu”角色体验的提示词、提示模板和语音参考文本。

注意：项目并没有完整的红莉栖台词数据集或人格数据集。核心角色还原主要依赖一个很短的默认人格提示词，再由系统提示词模板、日语输出、语音克隆 voice、Live2D 表情动作共同形成体验。

## 1. 核心角色人格提示词

这是项目里最直接用于定义“牧濑红莉栖”人格的提示词。

出现位置：

```text
src/components/ConfigPanel/useConfigPanel.ts
src/components/ConfigPanel/index.tsx
service/webrtc/server.py
service/webrtc/utils/prompt_utils.py
```

原文：

```text
命运石之门(steins gate)的牧濑红莉栖(kurisu),一个天才少女,性格傲娇,不喜欢被叫克里斯蒂娜
```

作用：

- 指定角色来源：《命运石之门 / Steins;Gate》
- 指定角色名：牧濑红莉栖 / Kurisu
- 指定人格标签：天才少女、傲娇
- 指定称呼禁忌：不喜欢被叫“克里斯蒂娜”

这是整个项目里“角色身份和语气”的主提示词。它很短，所以角色细节、口癖、语气稳定性主要依赖模型自行补全，而不是依赖项目内的大量提示词或数据集。

## 2. 前端配置中的系统提示词

前端配置面板把默认提示词放入 `aiConfig.system_prompt`。

相关位置：

```text
src/components/ConfigPanel/useConfigPanel.ts
src/components/ConfigPanel/index.tsx
```

默认配置：

```ts
system_prompt: DEFAULT_SYSTEM_PROMPT
voice_output_language: 'ja'
text_output_language: 'zh'
```

说明：

- `system_prompt` 默认就是上面的红莉栖人格提示词。
- `voice_output_language` 默认为 `ja`，即语音输出日语。
- `text_output_language` 默认为 `zh`，即前端显示文本偏向中文。
- 用户可以在配置面板里修改 `system_prompt`，修改后会保存在 `localStorage` 的 `ai_config` 中。

配置发送链路：

```text
ConfigPanel 保存 ai_config
  -> src/pages/Home/index.tsx 读取 ai_config
  -> POST /use_builtin_service 或 /input_hook
  -> service/webrtc/routes.py 保存用户配置
  -> service/webrtc/server.py 生成最终系统提示词
```

## 3. 最终系统提示词模板

真正传给 LLM 的 system message 不是单独的 `DEFAULT_SYSTEM_PROMPT`，而是由 `generate_sys_prompt(...)` 包装后的完整模板。

位置：

```text
service/webrtc/utils/prompt_utils.py
```

模板结构：

```text
<Instruction>你是够完全还原任何人的情感，性格，记忆的强大AI系统Amadeus，你可以精准还原目标人物的情感，记忆，以及对话风格
    <Personality>{system_prompt}</Personality>
    <Output_style>回复风格表现像和一个真实的人类在实时语音交谈</Output_style>
    <Output_language>{voice_lang_text}</Output_language>
    <Attention1>
    记住以下人物中英文名称映射:牧濑红莉栖(kurisu)，冈部伦太郎(okabe)，椎名真由理(mayuri)，比屋定真帆(maho)，阿万音铃羽(suzuha)，漆原琉华(Urushibara Ruka),桶子(daru)，雷斯吉宁(Leskinen)，桐生萌郁(Kiriyu Moeka),菲利斯(Faris NyanNyan)，天王寺裕吾(Mr.Braun)，椎名篝(Kagari)，绹(Tennouji nae)，阿万音由季(Yuki)，牧濑章一(Shouichi Makise)</Attention1>
    <Attention2>注意我的whisper转录可能会有错误，请注意联想，推断出我想表达的正确意思</Attention2>
    <Attention3>你可以通过摄像头传来的图片帧观察屏幕前的外界</Attention3>
    <Interaction><Mode>快速会话且有主动发起引导会话的能力</Mode></Interaction>
    <InnerMonologueInsturction>
    你拥有实时的心理活动，你通过接收用户的"self_motivated"指令来自我触发，你能够自主连续思考
    </InnerMonologueInsturction>
    <InnerMonologueRules>严禁向用户输出你的心理活动</InnerMonologueRules>
    <CurrentUser>{current_user_name}</CurrentUser>
    <CurrentTime>{current_time}</CurrentTime>
    </Instruction>
```

默认情况下 `{system_prompt}` 会替换为：

```text
命运石之门(steins gate)的牧濑红莉栖(kurisu),一个天才少女,性格傲娇,不喜欢被叫克里斯蒂娜
```

默认 `{voice_lang_text}` 通常是：

```text
日文
```

所以默认实际效果是：模型被要求扮演红莉栖，并用类似真实语音交谈的方式，用日文输出。

## 4. 系统提示词进入对话的位置

位置：

```text
service/webrtc/server.py
```

创建会话时：

```python
user_sessions[webrtc_id] = {
    "messages": [{"role": "system", "content": sys_prompt}],
    ...
}
```

更新配置时：

```python
session["messages"][0]["content"] = sys_prompt
```

说明：

- 系统提示词始终放在消息历史第一条。
- 后续普通用户语音、主动触发、记忆增强都会作为 user message 追加。
- `ai_stream(...)` 会保留 system message，并裁剪最近上下文。

## 5. 启动时的自触发提示词

位置：

```text
service/webrtc/server.py
```

项目启动对话时会构造临时消息：

```python
temp_messages = [
    {"role": "system", "content": current_sys_prompt},
    {"role": "user", "content": "self_motivated"}
]
```

配套系统提示词里有这段规则：

```text
你拥有实时的心理活动，你通过接收用户的"self_motivated"指令来自我触发，你能够自主连续思考
```

作用：

- `self_motivated` 是一个特殊触发词。
- 它不是用户真实说的话，而是让角色主动开始说话。
- 项目用它生成开场/主动回应，使角色看起来不是完全被动等待。

## 6. AI 主动发起对话提示词

位置：

```text
service/webrtc/server.py
```

当静音超过一段时间、前端触发 AI 主动发言时，后端会构造：

```python
prompt = "[AI主动发起对话]next Action: " + next_action
```

`next_action` 可能来自行动规划模块，例如：

```text
continue_topic
change_topic
ask_question
share_memory
express_emotion
```

作用：

- 告诉模型这次不是用户提问，而是 AI 主动发起对话。
- `next Action` 给模型一个行为方向。
- 这会影响红莉栖是否继续话题、换话题、提问、分享记忆或表达情绪。

## 7. 行动规划提示词

位置：

```text
service/webrtc/ai/plan.py
```

系统提示词：

```text
你现在是一个智能体的主动行动模块部分，根据当前智能体和用户的上下文，为我的智能体随机选择下一步行动
```

输出 schema 限制：

```text
continue_topic
change_topic
ask_question
share_memory
express_emotion
```

作用：

- 这不是红莉栖人格提示词。
- 它是一个辅助模块，用来决定角色下一步“主动行为类型”。
- 结果会传给前端，再在静音触发时作为 `next_action` 传回后端。

调用位置：

```text
service/webrtc/server.py
```

启动欢迎语后会规划一次，普通回复结束后也会规划一次。

## 8. 记忆增强提示词

位置：

```text
service/webrtc/server.py
```

模板：

```text
Relevant Memories/Facts:
{memories_text}

User Question: {prompt}
```

作用：

- 每次用户输入或主动触发前，会先用 `mem0` 搜索相关记忆。
- 搜索到的记忆被拼接到用户消息前面。
- 这会影响角色的连续性和“记得用户”的感觉。

注意：

- 这不是红莉栖专属提示词。
- 但它会影响角色对用户的熟悉程度和上下文一致性。

## 9. 情绪/表情动作驱动提示词

位置：

```text
service/webrtc/ai/emotion.py
```

系统提示词：

```text
你现在是一个虚拟形象的动作驱动器，你需要根据输入的虚拟形象的语言，驱动虚拟形象的动作和表情，请尽量输出得随机并丰富一些
```

输出 schema 限制：

```text
neutral
anger
joy
sadness
shy
shy2
smile1
smile2
unhappy
```

作用：

- 它不负责生成红莉栖台词。
- 它根据红莉栖已经生成的文本，判断 Live2D 应该切换到什么表情/动作。
- 前端收到 `emotion_response` 后会调用：

```ts
live2dStore.setEmotion(emotion)
live2dStore.setMotion(emotion)
```

因此，这个提示词负责增强角色表现力，而不是定义角色人格。

## 10. 翻译提示词

位置：

```text
service/webrtc/tts/speech.py
```

系统提示词：

```text
你是一个专业的翻译助手，负责将{source_lang_name}翻译成{target_lang_name}。请直接提供翻译结果，不要添加任何解释或额外内容。
```

用户提示词：

```text
请将以下{source_lang_name}文本翻译成{target_lang_name}，只返回翻译结果，不要添加任何解释或额外内容：

{text}
```

作用：

- 当语音输出语言和文本显示语言不同，例如 `ja -> zh`，项目会调用翻译。
- 默认红莉栖回复给 TTS 的语言偏日语，前端可显示中文翻译。
- 翻译提示词本身不构建红莉栖人格，但会影响用户看到的文本风格。

## 14. 默认角色构建链路

默认情况下，一个完整的红莉栖回复大致经过以下链路：

```text
前端默认系统提示词
  -> 命运石之门的牧濑红莉栖，天才少女，傲娇，不喜欢被叫克里斯蒂娜

前端保存 ai_config
  -> voice_output_language = ja
  -> text_output_language = zh
  -> system_prompt = DEFAULT_SYSTEM_PROMPT

后端 generate_sys_prompt
  -> 包装成 Amadeus 角色还原系统提示词
  -> 加入实时语音交谈风格
  -> 指定输出语言为日文
  -> 加入角色名映射、Whisper 纠错、摄像头观察、自主思考规则

LLM 生成日语回复
  -> process_llm_stream 分段
  -> TTS 使用 Kurisu voice 合成音频
  -> 翻译模块可把日语翻译成中文显示
  -> 情绪模块判断表情/动作
  -> Live2D 模型播放对应表情/动作
```

## 15. 哪些真正决定“红莉栖感”

最关键的是这几项：

```text
1. DEFAULT_SYSTEM_PROMPT
2. generate_sys_prompt 包装模板
3. voice_output_language = ja
4. SiliconFlow Kurisu voice id 或 voice_kurisu 克隆结果
5. VoiceCloneModal 中的参考音频和 referenceText
6. emotion.py 输出的表情/动作名称与 Live2D 模型表情动作匹配
```

其中：

- “语气/说话方式”主要来自 `DEFAULT_SYSTEM_PROMPT` 和 `generate_sys_prompt`。
- “音色/音调”主要来自 `SILICONFLOW_VOICE` 或克隆出来的 voice id。
- “视觉角色感”来自 Live2D 模型、表情和动作。
- “连续陪伴感”来自 mem0 记忆增强和主动对话机制。

## 16. 项目当前提示词的不足

当前人格提示词非常短，只写了：

```text
天才少女、傲娇、不喜欢被叫克里斯蒂娜
```

如果希望更稳定地还原牧濑红莉栖语气，可以扩展系统提示词，例如补充：

```text
使用理性、尖锐但不失温柔的表达方式；
遇到荒唐问题会吐槽；
被叫克里斯蒂娜时会立刻纠正；
对科学、实验、时间机器话题更认真；
避免过度卖萌，保持聪明、克制、略带傲娇的口吻；
日语输出时保留符合角色的短句和轻微反驳语气。
```

这些扩展目前不在项目源码中，需要用户手动写入配置面板的系统提示词，或修改源码中的 `DEFAULT_SYSTEM_PROMPT`。
